git bash 支持Linux的Git控制台 常用 git cmd 支持windows的Git控制台 git gui Git的可视化界面（使用很少） 本地的工作区 暂存区 远程仓库 查看自己绑定的用户： git config user.name git config user.email 修改自己绑定的用户名： git config --global user.name "problemer" git config --global user.email "1784012538@qq.com" 创建版本库：repository mkdir 创建空目录 cd 进入到创建的目录中 pwd 显示当前目录 git init 将该目录变成git可以管理的仓库（会在其中创建一个.git文件，该目录是用来进行跟踪管理版本库的） git init newrepo(指定目录作为git仓库) （.git目录是默认隐藏的，使用ls -ah命令可以看见） 和把大象放到冰箱需要3步相比，把一个文件放到Git仓库只需要两步。 把文件添加到仓库：（实际上是将要提交的所有修改放到暂存区） git add 文件名 把文件提交到仓库：（将暂存区中的修改提交到分支） git commit -m "提交本次文件的说明" commit可以一次提交很多文件，所以可以多次add不同的文件，一次性提交 继续修改我们的文件， 随时掌握工作区的状态:git status 如果git status告诉你有文件被修改过 使用 git diff 查看修改的内容 git log 显示最近到最远的提交日志、 可以试试加上--pretty=oneline 参数 版本回退： 退到上一个版本： git reset --hard HEAD^ (上上个版本就是HEAD^^ 往上很多个版本写不过来 可以写成HEAD~10 往上1 0个版本) 回退到某一指定版本（或者最新版本） git reset --hard fdsdsd(版本号，不用全写完，部分就好) 先用log命令查看要退回到那个版本，（当前窗口未关闭）去到未来的版本使用relog查看历史命令，确定回到的版本号。 git checkout -- 文件名 将文件在工作区的修改撤销 （分支管理，文件的恢复两个功能） git switch <分支名> git switch -b <分支名> git switch my-feature git restore 现在在做修改测试
